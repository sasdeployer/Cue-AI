# CLAUDE.md

Guidance for Claude Code (or any future contributor) working in this repo.

## What this is

Cue (repo/package name `Cue-AI`, on-page product name is just **"Cue"** — see
Branding below) is a v0.dev/Lovable-style product for **presentation decks**:
one prompt in, Cue generates a deck where every slide is a live, responsive
web page (the [bolt-slides](https://github.com/stackblitz/bolt-slides) engine).
Chat on the left, live preview on the right, full-screen when you want it.

## Architecture

| Layer | Tech | Notes |
|---|---|---|
| Frontend | React + Vite + TanStack Router | `web/` |
| Backend | Go + Gin | `server/`, streams generation over SSE |
| Database | Postgres + pgvector | `db/init.sql`, decks table (owner/is_public/embedding) |
| Deck engine | bolt-slides engine + ~30 component library | fixed; the model only authors `App.tsx` + `tokens.css` |

### Deck preview: same-origin hot-swap runtime (NOT Sandpack)

Sandpack (remote CodeSandbox bundler) was ripped out in Phase 1
(2026-07-21, see `docs/superpowers/specs/2026-07-21-phase1-hotswap-runtime-design.md`).
Current system:

- The fixed engine + component library are pre-baked into Cue's own bundle
  via `web/src/deck-runtime/registry.gen.ts` (generated by
  `scripts/gen-registry.mjs`, run automatically by `dev.sh`).
- Only the generated `App.tsx` is transpiled **in-browser** (Sucrase) and
  rendered in a **same-origin iframe** (`web/deck-runtime.html` /
  `runtime-entry.tsx`). `tokens.css` swaps via a `<style>` tag (no transpile
  needed for theme-only changes). Parent↔iframe talk over the `cue-*`
  postMessage protocol (`web/src/deck-runtime/protocol.ts`).
- `web/src/components/DeckPreview.tsx` hosts this iframe and is reused
  directly (unmodified) by `DeckThumbnail.tsx` for gallery card previews —
  rendered at a fixed 1200x750 stage size and shrunk via a
  `ResizeObserver`-driven CSS `transform: scale()`.
- Reliability: `server/compile.go` compile-checks the single generated file
  with esbuild-Go against a mirrored copy of the engine
  (`server/reference/engine`, synced by `dev.sh`) before it's ever shown to
  the user. Broken decks get an error overlay, never a blank pane.

### Generation pipeline + step feed

- `server/agent.go` runs an actual tool-using OpenAI agent loop (a
  `fetch_url` tool for light research), not just a single completion call.
- `server/handlers.go`'s `generateChecked` runs the model → parses →
  compile-checks → retries (≤3 attempts) on failure, and streams **real**
  pipeline steps as SSE `step` events the whole way (not decorative): think →
  write tokens.css → write App.tsx → check → (fix + retry if needed).
- Client: `web/src/lib/api.ts` dispatches `step` SSE events;
  `web/src/components/StepGroup.tsx` renders them live, merged by id.
  Labels are phrased cinematically ("Setting the scene", "Rolling tape") but
  map 1:1 to real steps — don't invent decorative-only labels here.

### LLM provider

`server/llm.go` behind an `LLMClient` interface — OpenAI, Anthropic, or a
`CannedClient` (no key needed, returns a static sample deck). Provider
priority in `server/main.go`: **OpenAI key present → OpenAI; else Anthropic
key present → Anthropic; else canned.** Model/env vars in `.env.example`.

**Known gotcha**: OpenAI's gpt-5-class models 400 if you pass function tools
without `reasoning_effort` explicitly set to `"none"` on `/chat/completions`
— omitting the field isn't enough, the API defaults to a non-none value.
Already handled in `llm.go`; don't regress this if touching the OpenAI client.

## Dev workflow

```bash
./dev.sh
```

Brings up Postgres+pgvector (Docker — **must be running**, `dev.sh` will hang
waiting for the container to be healthy otherwise), the Gin server (`:8080`),
and the web app (`:5273`). `dev.sh` also re-syncs the engine reference (skill
doc, tokens, component list, `server/reference/engine`) and regenerates
`web/src/deck-runtime/registry.gen.ts` on every start — this is why the deck
engine and the runtime's fixed module surface never drift apart.

Without any LLM key: canned mode, fully demoable. Real generation needs
`OPENAI_API_KEY` (or `ANTHROPIC_API_KEY`) in `.env` at repo root.

## Verification habits used in this repo

- `cd web && npx tsc --noEmit` and `cd server && go build ./...` after any
  edit — both are fast and catch real breakage before it's "done."
- Screenshot real behavior with Playwright (see prior sessions' scratch
  scripts) rather than trusting a description of what code *should* do —
  several real bugs this session (gpt-5 tool-call 400, a headline colliding
  with a photo's subject) were only caught this way.
- **Watch out for Mach banding**: a smooth CSS/SVG gradient can *look* like
  it has a banding seam in a screenshot preview when it doesn't actually
  exist in the pixel data. Before "fixing" a suspected gradient seam, sample
  raw pixel values (PIL `getpixel` diffing) to confirm it's real before
  spending time on it.
- Server-side edits require killing/restarting the `go run .` process
  manually (`lsof -ti:8080` → `kill -9` → relaunch) — it does not hot-reload.
  Frontend changes hot-reload via Vite automatically.

## Deferred / explicitly not built

- **Logo mark redesign** — several concepts sketched (cue light, spotlight-C,
  playhead) but none chosen. Current Aperture-icon badge is a placeholder,
  not a considered final mark. Treat as its own design session.
- **Gallery thumbnail toolbar bleed** — `DeckThumbnail` reuses the full
  interactive deck-runtime as-is, so its tiny nav toolbar renders faintly at
  the bottom of each card thumbnail (harmless — `pointerEvents: none` — but
  visible). Fixing it needs a "chrome-less" mode added to the runtime itself.
- **Phase 2**: Remotion video export, AI image generation.
- **Phase 3**: publish-to-Nexlayer, public-gallery cross-origin sandbox
  hardening (current same-origin trust boundary is a require-allowlist, fine
  for same-origin decks only).
- Auth/accounts, free/paid privacy toggle, billing, semantic gallery search
  UI (pgvector embeddings exist in the schema, no search UI built on top yet).
